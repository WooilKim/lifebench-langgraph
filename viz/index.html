📥 INPUT

📤 OUTPUT
// state.personas[0] {"name": "박준혁", "age": 46, "job": "개발자", "mbti": "INTJ", "relations": [15명]}
STEP 1
📅 Scheduler
5단계 순차 LLM 호출로 연간 ~190개 이벤트 생성. 카테고리별 비율 지정, 타임라인 충돌 해결. LLM ×5 중국어
根据{persona}的职业、性格、爱好，生成2025年100个生活事件。 类别比例: Career 25%, Personal Life 20%, Health 15%... 包含因果关系，避免冲突。JSON格式输出。
📥 INPUT
// persona.json {"name": "冯浩然", "job": "金融顾问"}
📤 OUTPUT
// daily_draft.json [{"date": "2025-02-01", "title": "春节家庭聚会", "category": "family"}, ... 190개 이벤트]
STEP 1
📅 draft_gen
한국 공휴일·직장 문화를 반영한 연간 이벤트 초안 생성. test 모드 10개 / 정식 40개. LLM ×1 한국어
박준혁(46세, 소프트웨어 개발자, 인천 거주)의 2025년 주요 생활 이벤트 40개를 JSON 배열로 생성하세요. 취미: 독서, 등산, 체스, 커피 원두 수집 한국 공휴일 반영: 설날(2025-01-29), 추석(2025-10-06)... 형식: [{"date","title","category","description"}] JSON만 출력하세요.
📥 INPUT
// state.personas[0] {"name": "박준혁", "job": "소프트웨어 개발자", "home_city": "인천"}
📤 OUTPUT
// state.daily_drafts["박준혁"] [{"date": "2025-01-29", "title": "설날 가족 모임", "category": "family"}, ... 40개 이벤트]
STEP 2
🎮 Mind (Simulator)
하루 7단계 LLM 호출: 주관적 계획 → 현실 제약 반영 → POI 매핑 → 이동경로 → 반성 → 장기기억. 365일 × max_workers=30 병렬. LLM ×7/day
【主观计划】{persona}今天的主观规划是什么？ 考虑：MBTI性格、今日事件、长期记忆... 【客观优化】考虑天气、交通、疲劳度... 输出：时间顺序事件序列 JSON
📥 INPUT
// persona + daily_draft {"date": "2025-01-29", "draft_event": "春节", "fuzzy_memory": "..."}
📤 OUTPUT
// daily_event.json [{"time": "07:00", "type": "起床", "location": "上海市徐汇区"}, ... 4,000~5,000개 총합]
STEP 2
🎮 simulator
이벤트 있는 날: LLM으로 5~8개 상세 이벤트. 평범한 날: 직장인 템플릿 자동 적용. test 모드에서는 전부 템플릿만 사용. LLM (이벤트 날만)
박준혁(개발자, 인천)의 아래 날짜 상세 이벤트를 JSON으로 생성하세요. - 2025-01-29: [family] 설날 가족 모임 출력: {"2025-01-29": [ {"datetime":"YYYY-MM-DD HH:MM:SS", "event_type":"...", "description":"...", "location":"..."} ]} JSON만 출력.
📥 INPUT
// state.daily_drafts {"date": "2025-01-29", "title": "설날 가족 모임"} // + 평일 템플릿 07:00 기상 → 09:00 출근 → 12:30 점심 → 18:00 퇴근
📤 OUTPUT
// state.daily_events_map["박준혁"] [{"datetime": "2025-01-01 07:00:00", "event_type": "기상", "location": "집"}, ... ~2,450개 이벤트]
STEP 3
📱 PhoneDataGenerator
daily_event.json 기반으로 7종 폰 데이터 생성. Communication·Push·NoteCalendar는 LLM, Gallery·Perception은 확률 기반. ThreadPoolExecutor(max_workers=7) 병렬. LLM + 확률
根据{date}的事件序列，生成通话记录： 事件: 和{contact}见面讨论工作 → 生成通话记录 JSON，包含时长、方向、号码... 注意：transcriptDialog 留空字符串
📥 INPUT
// daily_event.json (일별) {"date": "2025-01-02", "events": [{ "type": "工作会议", "participants": ["张伟"]}]}
📤 OUTPUT
// phone_data/call.json {"datetime": "2025-01-02 09:15:00", "contactName": "张伟", "direction": 0, "transcriptDialog": ""}
STEP 3
📱 phone_data_gen 규칙 기반
📞 call_gen
💬 sms_gen
🔔 noti_gen
🗂️ formatter
phone_data_gen: 이벤트에서 규칙 기반으로 raw 폰 데이터 합성 (LLM 없음)
call/sms/noti_gen: BehaviorEventEntity 변환 (원본 LifeBench 로직 유지)
formatter: 병합·정렬·contextGroupId 부여·페르소나별 분리
// 규칙 기반 로직 (phone_data_gen) 이벤트 유형 → 통화/문자 생성: social/family 이벤트 → 관계인과 통화 레코드 앱 알림: 취미(독서→예스24), 직업(개발자→GitHub) 한국 앱: 카카오톡, 토스, 배달의민족, 쿠팡... // BehaviorEventEntity 변환 (call_gen) direction=0 → type:0(발신), =1 → type:1(수신) "missed" in result → type:2(부재중)
📥 INPUT
// state.daily_events_map {"datetime": "2025-01-02 09:00:00", "event_type": "업무 시작", "location": "인천 사무실"}
📤 OUTPUT
// BehaviorEventEntity {"identifier": "call_0_1735689300000", "event_source": "CALL_LOG", "payload": { "name": "정우진", "type": 0, "duration": 120}, "contextGroupId": 1}
